◆ doHBlank()

void doHBlank ( int  line)

Perform horizontal blank processing for a given line.

This function is called during the horizontal blank period of the Game Boy's screen refresh cycle. It checks if graphics are disabled, the current line is outside the valid range, or if H-blank processing is disabled. If none of these conditions apply, it calculates the Game Boy line number for the current line, checks if the previous line was modified but not yet drawn, and draws the current line if it has been modified.

Additionally, if a file chooser or menu is currently active, this function will change the backdrop color for the current row to highlight the selected option.

Parameters
lineThe current line.

◆ drawScreen()

void drawScreen ( )

Draws the screen using the current rendering state.

If probingForBorder is true, the function immediately returns without performing any actions. Otherwise, if fast forward mode is not enabled, the function waits for vblank and sets the didVblank flag to false. If interruptWaitMode is 1, the function always waits for vblank, otherwise it waits only if vblank has not yet occurred. The frameFlip_Gameboy and frameFlip_DS fields of the sharedData struct are updated based on the current value of REG_VCOUNT. If gfxMask is nonzero, the function immediately returns without performing any further actions. The renderingState and drawingState pointers are swapped, and the screenDisabled variable is updated based on the value of the LCD control register. The tile data and tile maps are updated, and if SGB mode is active, the SGB palette is refreshed.

void loadSGBBorder ( )

Load SGB border.

This function sets up the background and loads the SGB border. It disables video background 3, sets the window out to video background 3, and sets the display control to mode 0. ◆ updateTiles()

void updateTiles ( )

Updates the tile data and queues the changed tiles to be redrawn.

The function updates the tile data and queues the changed tiles to be redrawn. It first processes all the tiles in the changedTileQueue, which holds the tiles that have changed on the current frame. After redrawing these tiles, it marks them as unmodified. Then, it copies the tiles in the changedTileInFrameQueue, which holds the tiles that have changed in the current frame but haven't been redrawn yet, to changedTileQueue so they can be redrawn on the next frame. Finally, it marks the tiles in the changedTileInFrame queue as unmodified.

◆ vblankHandler()

void vblankHandler ( )

Handle vblank interrupt.

This function is called during the vblank interrupt and performs various tasks such as setting the appropriate vram banks for scaling, capturing the main display into vram bank D, resetting the line completion status, and executing any vblank tasks that were added to the vblankTasks vector using doAtVBlank. If scaling is enabled, DMA copying is left for arm7 to handle and the scaleTransferReady flag is set to true. The function also updates the frame counter and the filter position if a filter is being applied.
